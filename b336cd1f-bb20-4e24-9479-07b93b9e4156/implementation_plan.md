# UI 세부 조정 및 가독성 최적화 계획

사용자의 요청에 따라 홈페이지의 시각적 균형과 가독성을 높이기 위한 세부 수정을 진행합니다.

## 주요 변경 사항

### 1. 게시물 날짜 오류 수정 (`AdminPage.jsx`)
- **수정**: `new Date()` 사용 시 UTC 기준이 아닌 한국 표준시(KST) 기준으로 날짜가 생성되도록 변경합니다. (`toLocaleDateString('ko-KR')` 활용)

### 2. 에디터 서식 적용 로직 전면 개편 (`RichTextEditor.jsx`)
- **문제**: 복잡한 `span` 태그와 인라인 스타일의 중첩은 HTML 구조를 쉽게 깨뜨리고 서식 변경을 방해합니다.
- **해결**: 브라우저 표준 `execCommand`의 기본 동작(font 태그)을 활용하여 로직을 단순화하고 안정성을 극대화합니다.
  - `styleWithCSS`를 끄고 `<font size="1-7">`, `<font face="...">` 태그를 사용하도록 변경합니다.
  - 7단계의 폰트 크기(1~7)를 각각 사용자가 원하는 픽셀 크기(10px~36px)로 매핑합니다.
- **글꼴**: 고딕, 명조, 지마켓 산스, 나눔스퀘어 4종을 `fontName` 명령을 통해 안정적으로 적용합니다.

### 4. 홈페이지 스타일 최적화 (CSS 매핑)
- **수정**: 관리자 페이지와 홈페이지 전역에 `<font size="1">`부터 `<font size="7">`까지의 실제 크기를 정의하는 CSS를 추가합니다.
- **효과**: 서식을 자주 바꾸더라도 태그가 꼬이지 않으며, 에디터에서 꾸민 그대로 홈페이지에 깨짐 없이 나타납니다.

## 검증 계획

### 수동 검증
1. **날짜 확인**: 새 글을 작성할 때 오늘 날짜가 정확히 나오는지 확인합니다.
2. **에디터 전면 확인**: 제목 칸에서도 글자를 굵게 하거나 색상을 바꿀 수 있는지 확인합니다.
3. **텍스트 깨짐 확인**: 서식을 입힌 제목과 본문이 홈페이지에서 깨지지 않고 의도한 대로(중앙 정렬, 색상 등) 나오는지 점검합니다.

## 검증 계획

### 수동 검증
1. **게시판 제목**: 홈페이지 하단 게시판 제목이 화면 중앙에 위치하는지 확인합니다.
2. **교회 소개**: 긴 텍스트가 있을 때 사진 높이를 넘어가지 않고 내부 스크롤이 생기는지 확인합니다.
3. **설명문 길이**: 각 섹션의 설명이 최대 두 줄까지만 표시되고 나머지는 생략되는지 점검합니다.

---
> [!NOTE]
> `line-clamp-2` 적용 시 텍스트가 잘릴 수 있으나, 이는 전체적인 디자인 통일성을 위한 조치입니다. 상세 내용은 각 페이지 이동을 통해 확인할 수 있습니다.
