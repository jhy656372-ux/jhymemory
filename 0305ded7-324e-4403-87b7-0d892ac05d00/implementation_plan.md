# 구현 계획 - 나의 사랑하는 책

## 목표
말씀 읽기(개역개정/NIV), CCM 찬양 듣기, 기도 제목 관리를 위한 아름답고 사용하기 편리한 웹 애플리케이션을 제작합니다.

## 사용자 검토 필요 사항
- **데이터 소스**:
  - **성경**: GitHub 등 오픈소스 저장소에서 개역개정(KRV) 및 NIV JSON 데이터를 확보하여 프로젝트 내 `src/data` 폴더에 포함시킵니다. 용량 최적화를 위해 책 별로 분할하거나 압축하여 로드합니다.
  - **찬양**: 유튜브 임베드 방식 유지.
- **데이터 저장**: 기도 제목은 별도의 서버 없이 사용자의 브라우저(LocalStorage)에 저장되어 유지됩니다.
- **오디오**: 성경 오디오 기능은 현재 디자인 UI만 구현되어 있으며, 실제 재생 기능은 추후 연동이 필요합니다.

## 변경 제안 사항

### 기술 스택
- **프레임워크**: React (Vite)
- **스타일링**: Tailwind CSS (나눔스퀘어네오/프리텐다드 + 본문용 명조체)
- **아이콘**: Lucide React
- **애니메이션**: Framer Motion
- **상태 관리**: React Context / Hooks

### 컴포넌트 구조

#### 레이아웃
- `App.jsx`: 메인 라우터 및 컨테이너.
- `Layout.jsx`: 네비게이션(모바일용 하단 탭, 데스크탑용 사이드바)을 포함한 전체 껍데기.

#### 주요 기능

**1. 말씀 (Bible)**
- `BiblePage.jsx`: 말씀 페이지 메인.
- `BibleReader.jsx`: 성경 본문 뷰어 (한/영 대조 기능).
- `ChapterSelector.jsx`: 성경 책/장 선택기.
- `AudioPlayer.jsx`: 오디오 재생 컨트롤 바.

**2. 찬양 (Praise)**
- `PraisePage.jsx`: 찬양 페이지 메인.
- `SearchBar.jsx`: 곡명/가수 검색창.
- `SongList.jsx`: 찬양 영상 리스트.
- `VideoPlayer.jsx`: 유튜브 영상 플레이어.

**3. 기도 (Prayer)**
- `PrayerPage.jsx`: 기도 페이지 메인.
- `PrayerInput.jsx`: 새 기도 제목 입력 폼.
- `PrayerList.jsx`: 기도 제목 리스트 (페이지네이션 포함).
- `Pagination.jsx`: 페이지 이동 버튼.

### 디자인 시스템
- **폰트**: 본문은 가독성 좋은 'Noto Serif KR', UI는 깔끔한 'Pretendard' 또는 'NanumSquareNeo'.
- **색상**:
    - 주조색: 부드러운 올리브 그린 / 웜 베이지.
    - 포인트: 차분한 골드 / 테라코타.
    - 배경: 미색 / 크림 화이트.

## 검증 계획
1. **말씀**: 한글/영어 버전 전환이 잘 되는지, 장 넘김이 자연스러운지 확인.
2. **찬양**: 검색어가 입력될 때 리스트가 필터링되는지, 영상이 재생되는지 확인.
3. **기도**: 5개 이상의 기도 제목을 입력하여 페이지가 잘 넘어가는지, 삭제 후 데이터가 유지되는지 확인.
4. **반응형**: 모바일과 데스크탑 화면에서 디자인이 깨지지 않는지 확인.

## 모바일 배포 계획 (Capacitor)
웹 애플리케이션을 네이티브 앱으로 패키징하기 위해 **Capacitor**를 사용합니다.

### 단계
1. **설치**: `@capacitor/core`, `@capacitor/cli`, `@capacitor/android` 설치.
2. **초기화**: `npx cap init`으로 앱 이름("나의 사랑하는 책")과 ID(`com.mybelovedbook.app`) 설정.
3. **빌드**: React 앱을 빌드하여 `dist` 폴더 생성.
4. **플랫폼 추가**: Android 플랫폼 추가 및 `dist` 폴더 동기화.
5. **설정 수정**:
    - `viewport-fit=cover` 메타 태그 추가 (노치 디자인 대응).
    - `capacitor.config.json` 설정 확인.
