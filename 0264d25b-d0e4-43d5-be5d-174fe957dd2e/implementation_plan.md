# UI 정밀 조정 및 여백 최적화 계획

## 목표
홈 화면은 기존의 부드러운(v25) 감성을 유지하고, 정보 전달이 중요한 서브 페이지(말씀, 찬양, 기도, 메모)는 공간 효율성을 극대화하여 내비게이션을 콘텐츠 박스에 붙이고 세로 여백을 줄입니다.

## 사용자 검토 필요 사항
> [!NOTE]
> - **홈 화면**: 내비게이션 바가 다시 둥근 캡슐 모양으로 떠 있게 됩니다 (v25 스타일).
> - **서브 페이지**: 내비게이션 바가 화면 하단에 꽉 차거나 콘텐츠 박스 너비에 맞춰지며, 상단 박스와 시각적으로 연결됩니다.
> - **전체적**: 제목 위/아래 여백, 리스트 간격 등이 좁아져서 더 많은 내용이 한 화면에 들어옵니다.

## 변경 제안

### 내비게이션 컴포넌트 (`Navigation.jsx`)
- `activeTab` prop에 따라 스타일을 분기합니다.
    - `home`: `rounded-[2rem]`, `mb-8`, `mx-4`, `shadow-xl` (Floating Style)
    - `others`: `rounded-none`, `mb-0`, `mx-0`, `border-t`, `shadow-none` (Flat Attached Style)

### 레이아웃 (`Layout.jsx`)
- `Navigation`을 감싸는 컨테이너의 스타일도 `activeTab`에 따라 조건부 적용합니다.
- 서브 페이지일 경우, 모바일 내비게이션 컨테이너의 `background`를 콘텐츠 영역과 자연스럽게 이어지도록 설정합니다.

### 페이지별 스타일 최적화
#### [MODIFY] 공통사항
- `Layout`의 `p-4`를 `p-2` 또는 `p-0`으로 줄이거나, 페이지 내부에서 제어하도록 수정하여 "밀착" 느낌을 구현합니다.
- `safe-area` 패딩은 유지하되, 불필요한 추가 `pt`나 `pb`를 제거합니다.

#### [MODIFY] `BiblePage.jsx`
- 상단 컨트롤 박스(`Top Box`)의 내부 패딩(`p-4` -> `p-2`) 감소.
- 제목 폰트 사이즈 및 마진 축소.
- 버전 선택 버튼의 패딩(`py-1.5` -> `py-1`) 감소.
- 내비게이션 박스와의 하단 여백 제거.

#### [MODIFY] `PraisePage.jsx`, `PrayerPage.jsx`, `MemoPage.jsx`
- 헤더 영역의 `py`, `mb` 감소.
- 리스트 아이템 간의 간격(`gap-3` -> `gap-2`) 축소.
- 불필요한 공백 제거.

## 검증 계획
### 수동 검증
1.  **홈 화면**: 내비게이션이 둥글게 떠 있는지 확인.
2.  **말씀 페이지**: 상단 컨트롤 박스가 콤팩트해졌는지, 하단 내비게이션이 직사각형으로 붙어 있는지 확인.
3.  **한 화면 가시성**: 스크롤 없이(또는 최소한의 스크롤로) 주요 내용이 잘 보이는지 확인.
