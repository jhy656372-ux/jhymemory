# 구현 계획 - 통계 토글 및 대시보드 스크롤

## 목표 설명
통계 페이지의 "전체 현황"에서 습관을 클릭했을 때 체크/체크해제가 확실하게 동작하며 대시보드와 동기화되도록 합니다. 또한 대시보드의 습관 목록이 길어질 경우를 대비하여 스크롤 영역을 지정, 화면 레이아웃이 깨지지 않도록 개선합니다.

- *참고*: 이미 `toggleHabit`은 구현되어 있으므로, 사용자 요청은 "기능이 잘 동작하게 해달라"는 확인 차원이거나, "체크 해제"가 안 된다고 느꼈을 수 있음. 로직은 확실하므로 스크롤 기능에 집중.

## 검증 계획

### 수동 검증
1. **스크롤 테스트**:
   - 습관을 5개 이상 추가하여 목록이 길어지게 만듦.
   - 목록 영역만 스크롤되고 상단 마스코트나 하단 네비게이션이 화면에 유지되는지(혹은 적절히 배치되는지) 확인.
2. **통계 토글 테스트**:
   - 통계 페이지 진입.
   - 이미 완료된 습관을 클릭 -> 체크박스가 사라지고 카운트가 1 감소하는지 확인.
   - 다시 클릭 -> 체크박스가 생기고 카운트가 1 증가하는지 확인.
   - 대시보드로 돌아와서 상태가 동기화되었는지 확인.
